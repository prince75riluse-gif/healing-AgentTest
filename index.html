⚠️ Skill Registry Testing: These tests simulate various exception scenarios that the Skill Registry should handle automatically during test execution.

Test Instructions

How to Test Skill Registry

  1. Setup: Enable useSkillRegistry: true in your project configuration
  2. Create Skills: Use the API to create skills for your scenario with matching trigger conditions
  3. Run Test: Execute your test scenario through Automator
  4. Trigger Interruption: Use the "Trigger" buttons on this page to simulate failures
  5. Observe: Skill Registry should detect, execute skill, and retry the step
  6. Verify: Check logs for skill execution details and MongoDB for execution records

🎯 Test 1: Ad Popup Interruption

Scenario Description

Test clicks the "Continue" button, but an ad popup blocks the click interaction.

Expected Exception: ElementClickInterceptedException
Skill Trigger: "When an unexpected ad popup appears on screen"
Skill Action: "Click on the close (X) icon"
Expected Outcome: Skill dismisses popup, step retries and succeeds

🍪 Test 2: Cookie Consent Banner

Scenario Description

Test clicks "Accept Terms", but cookie banner blocks the button.

Expected Exception: ElementClickInterceptedException
Skill Trigger: "When a cookie consent banner is blocking interactions"
Skill Action: "Click on the Accept All button"
Expected Outcome: Skill accepts cookies, step retries and succeeds

📧 Test 3: Newsletter Subscription Popup

Scenario Description

Test submits a form, but newsletter popup blocks submit.

Expected Exception: ElementClickInterceptedException
Skill Trigger: "When newsletter subscription popup appears"
Skill Action: "Click on the No Thanks button"
Expected Outcome: Skill dismisses newsletter, step retries and succeeds

👁️ Test 4: Element Not Visible (Scroll Required)

Scenario Description

Target button is outside viewport, causing ElementNotInteractableException.

Expected Exception: ElementNotInteractableException
Skill Trigger: "When target element is not visible in viewport"
Skill Action: "Scroll the target element into center of viewport"
Expected Outcome: Skill scrolls to element, step retries and succeeds

⏳ Test 5: Loading Spinner Blocker

Scenario Description

Loading spinner appears and blocks interaction, causing timeout.

Expected Exception: TimeoutException or ElementClickInterceptedException
Skill Trigger: "When loading spinner is blocking interactions"
Skill Action: "Wait for loading spinner to disappear"
Expected Outcome: Skill waits for spinner, step retries and succeeds

🔄 Test 6: Stale Element After DOM Update

Scenario Description

Element reference becomes stale after React/Vue-style re-render.

Expected Exception: StaleElementReferenceException
Skill Trigger: "When element reference becomes stale after DOM update"
Skill Action: "Wait 2 seconds for DOM to stabilize"
Expected Outcome: Skill waits, element re-located, step succeeds

🪟 Test 7: Multiple Overlapping Popups

Scenario Description

Three popups appear in sequence: Cookie → Newsletter → Promo

Expected Exception: ElementClickInterceptedException (multiple times)
Skill Trigger: "When any popup or modal is blocking interactions"
Skill Action: "Click close button or dismiss link"
Expected Outcome: Skill dismisses all popups, step eventually succeeds

  • ✅ Test 4: Element scrolled into view
  • ✅ Test 5: Waited for spinner to disappear
  • ✅ Test 6: Waited for DOM stabilization
  • ✅ Test 7: All popups dismissed in sequence
